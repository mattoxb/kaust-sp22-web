Python Sqlite

We’ve seen how to create tables in sqlite3, add data, and do some queries. We’ve also seen that it is a bit of a pain to do that using sqlite3’s native interface.

Now we are going to talk about how to use python to access sqlite3 directly.

The Library

Our language python has tons of libraries built in The python people like to say that their language has “batteries included”. , and we have one for sqlite3 as well. Here’s the syntax we’ll use to load it: 

import sqlite3 as sql

Normally we would use import sqlite3 by itself, which would mean that we would use syntax like sqlite3.connect() to access the library functions. This syntax allows us to say sql.connect() instead. Yes, we could have done the from sqlite3 import * but this time I wanted the sqlite3 commands to be separate from everything else.

Connecting to the Database

The first thing we need to do is connect to a database. We do this by creating a connection object. 

con = sql.connect('test.db')

This line creates a connection to a database that will be stored in the file test.db.

Cursors

We actually don’t use the connection object to interact much with the database; instead we create something called a cursor. 

cur = con.cursor()

We use a cursor to send a command or query to the database and retrieve the results. We might need more than one of these to be active at the same time; this is why we use cursors instead of using the connection object to do everything.

Creating Tables

If you have a program to interact with a database, you might need to have the program create the initial tables for you. We will set up a function that does this to keep all the creation materials in one place.

Let’s create a “todo” list manager. Step one: 

def setup():
    cur = con.cursor()
    cur.execute('create table tasks (name text, priority integer) ')
    cur.close()

setup()

If we run this once, you can see the file created. We can access it from the shell. (Demo)

What happens if we run this function a second time?

Only create tables once

The if not exists keyword fixes this for us. 

def setup():
    cur = con.cursor()
    cur.execute('create table if not exists tasks (name text, priority integer) ')
    cur.close()

setup()

Create data

To insert data, we need to create a string to send to the database. There are several ways to do this, but we will use this one today. This style is called qmark. We put a ? as a placeholder for where we want to add something, and use a tuple of values to fill it in. This works well for tables with just a few columns. 

def addTask(name,priority):
    cur = con.cursor()
    cur.execute('insert into tasks values (?,?)', (name,priority))
    cur.close()

Another function that sometimes comes in handy is executemany. 

def addHomework(name):
    hwList = [ ("Start homework " + name, 2),
               ("Finish homework " + name, 2),
               ("Turn in homework " + name, 1)
              ]

    cur = con.cursor()
    cur.executemany('insert into tasks values (?,?)',hwList)
    cur.close()

If you run these functions, open the database manually, try to select from the tables, you find there’s nothing in it!

You need to call cur.commit() when you are done making changes to save them. This gives you a way to abort a bunch of changes if something goes wrong. 

def addHomework(name):
    hwList = [ ("Start homework " + name, 2),
               ("Finish homework " + name, 2),
               ("Turn in homework " + name, 1)
              ]

    cur = con.cursor()
    cur.executemany('insert into tasks values (?,?)',hwList)
    cur.commit()
    cur.close()

Getting Information

We can use select and fetchrows() to get information from the database. 

def getTasks():
    cur = con.cursor()
    cur.execute('select * from tasks')

    for row in cur.fetchall():
        print("Task " + row[0] + " has priority " + row[1])

    cur.close()

You can also use row['task'] and row['priority'] here instead of indices.
